Version: 0.5.0
Date: ????
  Changes:
    - Helmod codec: map virtual recipes across the (energy / resource / boiler /
      fluid / rocket / agricultural / spoiling) lua_type discriminator instead
      of dropping every non-recipe production line. Best-effort: research,
      burnt, customized, and Helmod's pseudo-energy items still warn and drop
      because they have no factory_solver counterpart.
    - Helmod codec: omit the `factory` subtable on export of `spoiling`
      virtual recipes (and any future `is_support_factory = false` Helmod
      type), matching Helmod's own shape. Helmod's ModelCompute reads
      `recipe.factory.energy_total` unconditionally when the field exists
      and crashes if it's a placeholder; skipping the factory entirely
      lets Helmod's solver skip the per-line energy math the same way it
      does for its own spoilage recipes. On import we synthesize the
      `entity-unknown` sentinel machine factory_solver expects.
    - Helmod codec: surface a warning when an export produces zero recipes
      despite the source solution having production lines, and a per-line
      warning when a line is dropped because its `machine_typed_name` is
      missing. Validates the serialised model round-trips through
      `helpers.encode_string` / `loadstring` before returning so a malformed
      payload is caught here instead of as an opaque Helmod-side crash.
    - Helmod codec: substitute the agricultural-tower for the plant entity
      in `<grow>` virtual recipe exports. factory_solver uses the plant
      itself as the line's machine (it occupies a tower slot), but Helmod's
      EntityPrototype:getCraftingSpeed dereferences energy_source_prototype
      and crashes when fed a plant entity. The plant identity is preserved
      in the FS recipe name (`<grow>{plant}:{seed}`) and restored on import.
    - Helmod codec: drop thruster virtual recipes (`<run>{thruster_entity}`)
      with a warning instead of exporting them as Helmod `lua_type = "energy"`.
      Helmod's getEnergyMachines filter excludes the thruster prototype type,
      so the entry would otherwise deserialize into a 0-W / 0-factory ghost
      recipe with no meaningful semantics on Helmod's side.
    - Helmod codec: drop import constraints / objectives whose Helmod-side
      entry has `type="energy"` (the electricity / steam-heat pseudo-item).
      factory_solver's LP has no electricity channel, so these were
      previously silently coerced into "item" constraints and surfaced in
      the UI as unknown-item "?" placeholders.
    - Helmod codec: skip import of `factory.fuel = "steam-heat"` or
      `"energy"` so the preset lookup fills in factory_solver's `<heat>`
      virtual_material for heat-exchanger and similar heat-energy machines
      instead of producing an unresolved "?" fuel slot.
    - Helmod codec: never emit factory_solver virtual_material names (e.g.
      `<heat>`) into the exported `factory.fuel` field. Helmod models heat
      energy as a recipe ingredient (`steam-heat` pseudo-item), not as a
      `factory.fuel` value, so writing the virtual name caused Helmod to
      round-trip it verbatim and corrupted later imports with an unknown-
      item fuel slot. Defensive `<`-prefix detection on the import side
      keeps older exports recoverable.
    - Helmod codec: fall back to `accessor.try_get_fixed_fuel` and then to
      the user's FS fuel preset when import can't resolve `factory.fuel`
      for a fuel-using machine. Heat-energy machines (which Helmod stores
      without factory.fuel) take the fixed-fuel branch; generic burners
      (stone-furnace, pY's burner family, ...) take the preset branch.
      Without these fallbacks the imported line entered storage with a
      nil fuel_typed_name and tripped `normalize_production_line`'s
      assert during the first solve.
    - Helmod codec: dedupe import production lines that resolve to the
      same (type, name, quality) tuple. Helmod's hierarchical model can
      hold the same recipe in multiple blocks and our flattening turned
      that into duplicate LP variables, tripping `add_objective`'s "same
      name" assert. Subsequent occurrences are dropped with a warning.
    - Helmod codec: aggregate the per-recipe `beacon per_factory`, module-
      priority, and Helmod-specific-constraints import warnings into one
      counted line each. These fire on every recipe that uses the feature
      and otherwise drown out the rare per-line warnings (duplicate
      recipe, production multiplier ≠ 1, unmappable virtual recipe).
    - Factory Planner codec: surface previously-silent feature drops.
      Import: warn on subfloor flattening, on per-line `percentage` ≠
      100%, on inactive lines being imported as active, and on dropped
      `priority_product` selection (the last three aggregated as one
      counted line each, since the recipe identity isn't actionable).
      Export: warn on upper / equal item-fluid bounds (FP's product
      target has produce-at-least semantics) and on dropped
      temperature constraints (FP has no fluid-temperature dimension).
    - Route quality multiplier calculations through Factorio 2.0's per-prototype
      quality runtime APIs (LuaQualityPrototype::default_multiplier,
      LuaEntityPrototype::get_max_energy_usage / get_fluid_usage_per_tick /
      get_pumping_speed / get_researching_speed / get_crafting_speed at a given
      quality) instead of assuming the vanilla 1 + 0.3 x tier constants. Per-second
      throughputs for virtual recipes (boilers, reactors, generators, fusion-*,
      thrusters, offshore-pumps, mining, plants, labs) are now derived at solve
      time from the engine. Vanilla numerics are preserved; mods that customize
      QualityPrototype multipliers or per-machine quality scaling are now honored.
    - Cap quality cascade decomposition at utility-constants
      maximum_quality_jump tiers above the source quality. Vanilla default is
      effectively unlimited; mods that lower the cap are now respected.

Version: 0.4.2
Date: 2026-05-26
  Gui:
    - Show a "No configurable items." placeholder in the machine settings
      dialog for spoilage virtual recipes, which have nothing to configure.
  Changes:
    - Keep beacon configuration in storage when the chosen machine cannot
      receive beacon effects, so it reappears intact if a beacon-capable
      machine is selected later.
    - Drop the per-recipe negative cost from the LP objective; reachability
      gating, elastic sinks, active-line pruning, and the new IPM together
      cover its stabilisation role.
    - Charge a small cost on external material sources so the solver prefers
      the chain drawing less raw input over arbitrary splits.
  Bugfixes:
    - Apply the beacon's quality tier to its distribution_effectivity, so
      higher-quality beacons boost module effects by the engine-side amount
      (vanilla: base + 0.2 per quality level).
    - Fix lower and equal constraints being silently violated when the
      producer chain emits many unclearable by-products.
    - Fix the Substrate section being hidden in the machine settings dialog
      for plant production lines (regression from 0.4.1).

Version: 0.4.0
Date: 2026-05-23
  Features:
    - Solve quality recycling loops (Space Age) without manual unrolling. (Thank you, Claude!)
    - Solve recipe chains dominated by byproducts (e.g. Fulgora recycling) by
      gating elastic shortage sources on reachability from raw inputs, so the
      solver can no longer bypass producer chains by paying elastic cost.
    - Add a fluid temperature dimension: boilers, heat exchangers, generators,
      and fusion reactors are split per (fluid, temperature) and bridged automatically.
    - Add a Research Bonus snapshot dialog that folds force productivity bonuses
      into the solver and unlocks qualities. Qualities above normal start locked,
      so their checkboxes must be enabled here before quality modules have any
      effect on the quality cascade.
    - Add virtual recipes for fusion reactors, fusion generators, thrusters,
      agricultural towers, labs (dispatched per consumed pack), per-fluid boilers,
      and item spoilage.
  Changes:
    - Rewrite the interior-point solver as long-step path-following with cold-start
      scaling, relative residuals, and retry-on-NaN Cholesky for stable convergence
      on deep quality recycling cascades.
    - Auto-promote cycle entry materials to basic sources via SCC analysis so that
      mass-losing loops with no external input still have a bootstrap.
    - Prune production lines isolated from any Constraint out of the LP.
    - Dispatch offshore-pump virtual recipes by fluid name instead of a fixed machine.
  Gui:
    - Surface bare-fluid temperature ranges in the constraint and fuel pickers, and
      bind bare-fluid constraints to every temperature variant.
    - Use the native sprite-button quality indicator overlay.
    - Tear down open GUIs on configuration change to avoid stale references.
    - Display the launched-rocket virtual material as a cargo pod.
    - Pick the initial filter group deterministically from prototype order.
    - Decompose production line slots, totals, and constraints by quality tier.
  Bugfixes:
    - Fix crash when a production line references a removed prototype.
    - Fix tile and resource virtual recipes always reporting as researched, so
      planet-locked materials no longer appear in the picker before the host
      space location is discovered.
    - Fix constraint lookup ignoring quality in solution settings.
    - Fix totals double-counting phantom internal flows from the quality cascade.
    - Fix IPM instability when the constraint RHS is sub-unit.
    - Fix module quality effect ignoring the current tier's next_probability.
